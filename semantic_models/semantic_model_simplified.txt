# === CRITICAL: Never filter on sale_amount or return_amount ===
# NEVER add WHERE sale_amount > 0 or WHERE return_amount > 0.
# Zeros are legitimate data — filtering them corrupts aggregation denominators.
# For return_rate: a product sold 100 times but returned once has return_rate = 1%.
# If you filter WHERE return_amount > 0, you keep only the 1 returned row and get 100%.
# Always aggregate ALL rows. Let SUM() handle the zeros.
#
# Exception — use row_type to distinguish sale vs return rows inside COUNT:
#   sales_tickets  = COUNT(DISTINCT CASE WHEN row_type = 'sale'   THEN ticket_number END)
#   return_tickets = COUNT(DISTINCT CASE WHEN row_type = 'return' THEN ticket_number END)
# Do NOT use sale_amount > 0 as a proxy — sale rows can have sale_amount = 0 (NULL price/qty).
# Do NOT use COUNT(DISTINCT ticket_number) alone — counts both sale and return rows.
